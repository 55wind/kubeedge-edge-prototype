# Edge Auth Manager Dashboard 사용 가이드

## 접속 방법

브라우저에서 아래 주소로 접속합니다.

```
http://<cloud VM IP>:30501
```

> 현재 환경 기준: `http://172.26.171.10:30501`

---

## 페이지 구성

대시보드는 좌측 사이드바를 통해 8개 페이지로 이동할 수 있습니다.

| # | 페이지 | 용도 |
|---|--------|------|
| - | Home | 빠른 상태 요약 및 각 페이지 바로가기 |
| 1 | Overview | 시스템 전체 건강 상태 |
| 2 | Devices | 디바이스 등록/승인/폐기 관리 |
| 3 | Authentication | 인증 성공/실패 분석 |
| 4 | API Performance | API 응답 속도 및 에러율 |
| 5 | Security | 보안 인시던트 관리 |
| 6 | Logs | API 요청 로그 조회 |
| 7 | Checklist | 개발 요구사항 검증 체크리스트 |
| 8 | Test Scenarios | 자동화된 통합 테스트 실행 |

## 1. Overview (시스템 개요)

시스템 전체 상태를 한눈에 파악하는 페이지입니다.

**표시 정보:**
- 디바이스 요약: 전체/승인/대기/폐기 수
- 24시간 요청 수, 인증 성공률
- 활성 보안 인시던트 수
- 시간대별 요청량 추이 차트
- 디바이스 상태 분포 파이 차트
- 시스템 가동시간, 평균 지연시간, 건강 상태
- 보안 설정 요약 (mTLS, JWT TTL, RBAC 역할 수 등)

**사용법:**
- 상단의 **시간 범위**(1h/6h/24h/7일)를 선택하여 조회 기간을 변경합니다.
- **Auto-refresh**를 켜면 30초마다 자동 갱신됩니다.

---

## 2. Devices (디바이스 관리)

엣지 디바이스의 전체 라이프사이클을 관리하는 핵심 페이지입니다.

**표시 정보:**
- 디바이스 목록 (상태별 색상 구분: 노란색=대기, 초록색=승인, 빨간색=폐기)
- 선택한 디바이스의 인증서 상태 및 만료일
- 비활성 디바이스 목록 (24시간 이상 미접속)
- 인증서 폐기 목록 (CRL)

**주요 기능:**

| 버튼 | 동작 | 설명 |
|------|------|------|
| Approve | 디바이스 승인 | PENDING 상태의 디바이스를 APPROVED로 변경 |
| Revoke | 디바이스 폐기 | 디바이스를 REVOKED 상태로 변경 |
| Issue Cert | 인증서 발급 | 선택한 디바이스에 mTLS 인증서 발급 |
| Renew Cert | 인증서 갱신 | 기존 인증서를 새 인증서로 교체 |
| Decommission | 디바이스 해제 | 디바이스를 시스템에서 완전 제거 |
| Transfer | 디바이스 이전 | 다른 사이트/그룹으로 디바이스 이동 |

**사용 순서 (신규 디바이스):**
1. 상태 필터에서 **PENDING** 선택 → 대기 중인 디바이스 확인
2. 디바이스 선택 → **Approve** 클릭
3. **Issue Cert** 클릭하여 인증서 발급
4. 디바이스가 정상 통신 시작

---

## 3. Authentication (인증 분석)

인증 시도의 성공/실패를 분석하는 페이지입니다.

**표시 정보:**
- 총 인증 시도 횟수, 성공/실패 건수, 성공률
- 시간대별 인증 성공/실패 추이 (누적 막대 차트)
- 실패 원인 분류 파이 차트
- 디바이스별 인증 통계 테이블
- JWT 보안 설정 (알고리즘, TTL, 차단된 토큰 수)
- RBAC 권한 매트릭스

**주요 기능:**
- **실패율 분석**: 실패율에 따라 자동으로 경고 메시지 표시
  - 10% 이상: 빨간색 경고
  - 5% 이상: 노란색 주의
  - 5% 미만: 초록색 정상
- **OCSP 조회**: 특정 디바이스의 인증서 유효 상태를 실시간 확인
  - 디바이스 namespace를 입력하면 GOOD/REVOKED/UNKNOWN 상태 표시

---

## 4. API Performance (API 성능)

Manager API의 응답 속도와 에러율을 모니터링합니다.

**표시 정보:**
- 총 요청 수, 5xx 에러 수, 에러율
- 응답 지연시간: P50 (중앙값), P95, P99
- 시간대별 요청량 및 에러 추이 차트
- 엔드포인트별 에러율 막대 차트
- 엔드포인트별 상세 통계 테이블

**건강 상태 기준:**
- 지연시간: < 100ms (우수) / < 500ms (양호) / < 1s (보통) / > 1s (불량)
- 에러율: 0% (우수) / < 1% (양호) / < 5% (보통) / > 5% (불량)

> 에러율 5% 초과 또는 P99 > 1초일 경우 개선 권고사항이 자동 표시됩니다.

---

## 5. Security (보안 인시던트)

보안 관련 이벤트를 심각도별로 관리합니다.

**인시던트 유형:**
- `AUTH_FAILURE_BURST` - 인증 실패 급증
- `STALE_DEVICE` - 장시간 미접속 디바이스
- `CERT_EXPIRED` - 인증서 만료
- 기타 보안 이벤트

**심각도 분류:** CRITICAL / HIGH / MEDIUM / LOW

**사용법:**
1. 상단 카드에서 심각도별 인시던트 수 확인
2. CRITICAL/HIGH 인시던트는 자동으로 펼쳐서 표시됨
3. 각 인시던트 카드에서 **Resolve** 버튼으로 해결 처리
4. **Show resolved** 체크박스로 해결된 인시던트 표시/숨김

---

## 6. Logs (API 로그)

API 요청 로그를 조회하고 분석합니다.

**필터 옵션:**
- 시간 범위: 1h / 6h / 24h / 7일
- 최대 표시 건수: 50 / 100 / 200 / 500
- 상태 코드 필터: All / 2xx / 4xx / 5xx
- 경로 검색: 특정 API 경로 포함 필터

**로그 테이블 색상:**
- 초록색: 2xx (정상)
- 노란색: 4xx (클라이언트 오류)
- 빨간색: 5xx (서버 오류)

**CSV 내보내기:** 필터링된 로그를 CSV 파일로 다운로드할 수 있습니다.

---

## 7. Checklist (요구사항 체크리스트)

프로토타입 개발 요구사항의 구현 상태를 확인합니다.

**4가지 요구사항 영역:**
1. 경량 에이전트 보안 모듈 (등록, 탐색, 메타데이터 전송)
2. Manager API (디바이스 승인, 인증서 관리, 토큰 처리)
3. 엣지 간 통신 (RabbitMQ 메시지 버스)
4. 보안 및 컴플라이언스 (mTLS, JWT, RBAC, CRL/OCSP)

**검증 방식:**
- **대시보드 검증 가능**: 해당 페이지로 이동하는 버튼 제공
- **코드 검증 필요**: 소스 코드 스니펫과 설명 제공

---

## 8. Test Scenarios (자동화 테스트)

4개 요구사항 영역을 자동으로 테스트하는 시스템입니다.

**테스트 카테고리 (총 35+ 테스트):**

| 카테고리 | 테스트 수 | 내용 |
|----------|-----------|------|
| Agent Security | 7 | 디바이스 등록, 중복 감지, 응답 구조 검증 |
| mTLS/Session | 6 | mTLS 연결, 세션 재사용, 동시 요청, 지연시간 |
| Auth Module | 15+ | 승인, 토큰 발급/검증, 인증서 수명주기, RBAC |
| Message Bus | 7 | RabbitMQ 상태, 큐, TLS 연결, 권한 제한 |

**사용법:**

1. **단일 실행**: `Run Single Cycle` 클릭 → 전체 테스트 1회 실행
2. **연속 실행**:
   - 테스트 간격 설정 (10~300초)
   - `Start Continuous Testing` 클릭
   - 자동으로 반복 실행되며 이력이 누적됨
   - `Stop` 버튼으로 중지

**결과 확인 탭:**
- **Current Cycle**: 현재 실행 결과 (PASS/FAIL/ERROR 색상 구분)
- **Historical Stats**: 통과율 추이 차트 및 사이클별 이력
- **Category Breakdown**: 카테고리별 통과율
- **Test Log**: 전체 테스트 로그 (필터 가능)
- **Security Config**: 현재 보안 설정 표시

## 공통 기능

### 시간 범위 선택
대부분의 페이지에서 조회 기간을 선택할 수 있습니다:
- **1h**: 최근 1시간
- **6h**: 최근 6시간
- **24h**: 최근 24시간 (기본값)
- **7 days**: 최근 7일

### Auto-refresh
토글을 켜면 **30초마다** 데이터가 자동 갱신됩니다.
실시간 모니터링 시 유용합니다.

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 페이지 로딩 안 됨 | Dashboard Pod 다운 | `kubectl get pods -n edge-auth -l app=dashboard` 확인 |
| 데이터 표시 안 됨 | Manager API 연결 실패 | Manager Pod 상태 및 인증서 확인 |
| 디바이스 목록 비어있음 | Agent 미등록 | Agent Pod 로그 확인 (`/var/log/pods/` 에서) |
| 테스트 전체 실패 | mTLS 인증서 문제 | Secret `admin-certs`, `agent-certs` 존재 여부 확인 |
| 접속 타임아웃 | NodePort 미도달 | cloud VM IP 변경 여부 확인 |
